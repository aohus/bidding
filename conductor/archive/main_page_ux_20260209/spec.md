# Specification: 메인 페이지 UX/UI 개선

## 1. Overview
사용자가 서비스를 랜딩했을 때의 초기 경험을 개선하고, 검색 효율성을 높이기 위해 메인 페이지의 기본 동작, 필터 UI, 캐싱 및 다중 검색 로직을 고도화합니다.

## 2. Functional Requirements

### 2.1 기본 데이터 로딩 및 상태 유지
- **자동 로딩:** 페이지 첫 진입 시 '개찰일시' 기준, '오늘 날짜(00:00 ~ 23:59)' 범위로 자동 검색을 수행합니다.
- **상태 캐싱:** 사용자가 마지막으로 수행한 검색 조건과 결과 데이터를 `LocalStorage`에 저장합니다. 브라우저를 새로고침하거나 재방문했을 때 이전 상태를 즉시 복원합니다.

### 2.2 검색 필터 UI 개선
- **날짜 선택기 통합:** 시작일과 종료일을 각각 선택하던 방식에서, 하나의 달력 컴포넌트 내에서 기간(Range)을 선택하는 방식으로 변경합니다.
- **다중 검색 입력 (Tag UI):** '참가제한지역명' 및 '업종명' 필드에 태그 형태의 입력 UI를 적용합니다. 사용자가 콤마(,) 또는 엔터 키를 입력하면 개별 태그로 분리됩니다.

### 2.3 다중 API 호출 로직
- '참가제한지역명' 또는 '업종명'에 여러 개의 태그가 입력된 경우, 백엔드 또는 프론트엔드에서 각 조건별로 API를 병렬 호출하여 결과를 합산하고 중복을 제거한 후 표시합니다.

### 2.4 검색 조건 관리
- **불러오기 모달:** 사용자가 저장한 검색 조건 목록을 별도의 모달 창에서 확인하고 선택하여 즉시 적용할 수 있는 기능을 제공합니다.

## 3. Technical Requirements
- **Frontend:**
  - `LocalStorage`를 활용한 상태 관리.
  - 기간 선택을 위한 `shadcn/ui`의 Date Range Picker 활용.
  - 태그 입력을 위한 전용 컴포넌트 구현.
- **Backend:**
  - 프론트엔드에서의 병렬 `fetch` 처리를 지원하도록 기존 검색 API 활용.

## 4. Acceptance Criteria
- [ ] 페이지 진입 시 오늘자 개찰 공고가 자동으로 표시되는가?
- [ ] 새로고침 후에도 이전 검색 결과와 필터 값이 유지되는가?
- [ ] 하나의 달력에서 시작/종료일을 선택할 수 있는가?
- [ ] 지역명이나 업종명을 여러 개 입력했을 때 합산된 결과가 정상적으로 출력되는가?
- [ ] 저장된 검색 조건을 모달에서 선택하여 불러올 수 있는가?
